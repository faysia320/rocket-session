# 작업 이력: 사이드바 버튼 정리 (네비게이션 분리)

- **날짜**: 2026-02-21
- **작업자**: Claude + 사용자
- **브랜치**: main

## 변경 요약

사이드바 하단에 성격 구분 없이 나열되어 있던 9개 버튼을 역할별로 분리 정리했습니다.
페이지 이동 버튼(Dashboard, History, Analytics)을 사이드바 상단 네비게이션으로 이동하고,
하단 Footer의 나머지 6개 버튼을 설정 그룹과 레이아웃 그룹으로 시각적으로 분리했습니다.

## 변경 파일 목록

### Frontend

- `frontend/src/features/session/components/Sidebar.tsx` - 사이드바 레이아웃 재구성

## 상세 변경 내용

### 1. 상단 네비게이션 섹션 추가

- Dashboard, History, Analytics 3개 페이지 이동 버튼을 사이드바 최상단 `<nav>` 영역으로 이동
- Expanded 상태: 수평 균등 배치 (아이콘 + 텍스트), 현재 페이지 `bg-muted` 강조
- Collapsed 상태: 세로 아이콘만 배치, Tooltip (side="right")
- `isHistory` 변수 추가로 History 페이지 활성 상태 강조 지원

### 2. Footer 버튼 재구성 (9개 → 6개, 2그룹)

- 기존: [알림][설정][템플릿][히스토리][테마][대시보드][분석][분할뷰][접기] (9개, 구분 없음)
- 변경: [알림][설정][템플릿] │ [테마][분할뷰][접기] (6개, 구분선으로 분리)
- 설정 그룹: 알림 토글, 글로벌 설정 Dialog, 세션 템플릿 Dialog
- 레이아웃 그룹: 테마 토글, Split View 토글, 사이드바 접기/펼치기
- 구분선: Expanded에서 세로선(`w-px h-5`), Collapsed에서 가로선(`h-px w-5`)
- 모바일: 구분선 숨김, 레이아웃 그룹 중 분할뷰/접기 숨김 (기존 동작 유지)

### 3. New Session 영역 간격 조정

- 상단 네비게이션 추가로 인해 New Session 섹션의 상단 패딩을 `pt-3` → `pt-2`로 축소

## 테스트 방법

1. 사이드바 상단에 Dashboard/History/Analytics 탭이 표시되는지 확인
2. 각 탭 클릭 시 해당 페이지로 이동, 현재 페이지 강조 표시 확인
3. 사이드바 접기(collapsed) 상태에서 아이콘 + Tooltip 동작 확인
4. Footer 하단에 설정/레이아웃 2그룹 구분선 표시 확인
5. 모바일에서 상단 네비 표시, Footer 레이아웃 그룹 부분 숨김 확인
