# 작업 이력: 모바일 대시보드 탭 레이아웃

- **날짜**: 2026-03-05
- **작업자**: Claude + 사용자
- **브랜치**: main

## 변경 요약

모바일(767px 이하)에서 대시보드의 세션 현황 카드와 히스토리가 세로로 길게 스택되어 사용성이 떨어지는 문제를 해결. 모바일에서만 "세션 현황" / "히스토리" 탭 UI로 전환하여 각 영역을 독립적으로 볼 수 있도록 개선.

## 변경 파일 목록

### Frontend

- `frontend/src/routes/index.tsx` - 모바일 탭 레이아웃 분기 추가

## 상세 변경 내용

### 1. 모바일 탭 UI 추가

- `useIsMobile()` 훅으로 767px 이하 감지
- shadcn/ui `Tabs` 컴포넌트(Radix UI 기반, 기존 설치됨)를 사용하여 2개 탭 구성
  - **세션 현황** 탭: `DashboardGrid` (기본 활성)
  - **히스토리** 탭: `HistoryPage`
- `defaultValue="dashboard"`로 탭 상태 관리 (로컬, URL/스토어 불필요)

### 2. 데스크톱 레이아웃 유지

- `if (!isMobile)` 분기로 기존 세로 스택 레이아웃 완전 보존
- 데스크톱 사용자에게 영향 없음

### 3. 핸들러 리팩토링

- `handleSelect`, `handleNew`를 함수로 추출하여 모바일/데스크톱 분기에서 재사용
- 인라인 콜백 중복 제거

### 4. Empty state 유지

- 활성 세션 0개인 경우 탭 없이 기존 EmptyState + HistoryPage 레이아웃 그대로 유지

## 테스트 방법

1. 브라우저 DevTools에서 767px 이하로 리사이즈
2. "세션 현황" / "히스토리" 탭 전환 동작 확인
3. 768px 이상으로 리사이즈 시 기존 세로 스택 레이아웃 복원 확인
4. 활성 세션 0개 → 탭 없이 EmptyState 표시 확인
