# 작업 완료 체크리스트

## PR 전 필수 확인
- [ ] 타입 에러 없음 (`vue-tsc --noEmit`)
- [ ] `pnpm lint:xxx` 통과
- [ ] 관련 테스트 통과
- [ ] 비밀값 미포함
- [ ] 파일 ≤ 300 LOC, 함수 ≤ 50 LOC 준수

## 변경 전 확인 사항
1. 파일은 **처음부터 끝까지** 철저히 읽기
2. 수정할 파일의 **import/export 관계** 파악
3. **정의, 참조, 호출 지점, 관련 테스트, 문서/설정** 확인
4. 해당 함수/컴포넌트의 **사용처** 전역 검색
5. 결정 전 **최소 두 가지 대안 비교**

## 테스트 원칙
- **새 기능**: 테스트 필수
- **버그 수정**: 실패 테스트 먼저 작성 → 수정
- **위치**: 소스 파일과 같은 폴더에 `.test.ts`
- **커버리지 목표**: 80% (핵심 로직 100%)

### 테스트 생략 가능 예외
- 프로토타입 또는 POC
- 단순 UI 스타일링 수정 (로직 변경 없음)
- 긴급 핫픽스 (단, 사후에 반드시 테스트 추가)

## 핵심 사이클
**문제 정의 → 작고 안전한 변경 → 변경 리뷰 → 리팩터링** — 이 루프를 반복

## 데이터 무결성 원칙
기존 데이터 삭제/수정은 새 작업이 성공한 후에만 수행
